## 問1

情報セキュリティの三要素（CIA）に含まれないものはどれか。

1. 機密性（Confidentiality）
2. 完全性（Integrity）
3. 可用性（Availability）
4. 信頼性（Reliability）


<QuizAnswer>

**答え: 4**

情報セキュリティの三要素（CIA）は、機密性（Confidentiality）、完全性（Integrity）、可用性（Availability）の3つです。信頼性（Reliability）はシステムの品質に関する概念であり、CIAには含まれません。JIS Q 27001（ISMS）でも、この3要素を維持することが情報セキュリティの定義とされています。

</QuizAnswer>

## 問2

コンピュータに保存されたファイルを暗号化して使用できなくし、復号と引き換えに金銭を要求するマルウェアはどれか。

1. トロイの木馬
2. ワーム
3. ランサムウェア
4. スパイウェア


<QuizAnswer>

**答え: 3**

ランサムウェア（Ransomware）は、「Ransom（身代金）」と「Software」を組み合わせた造語で、感染するとファイルを暗号化し、復号のための金銭（身代金）を要求するマルウェアです。IPAの「情報セキュリティ10大脅威」でも毎年上位にランクインしており、組織・個人ともに大きな脅威となっています。

</QuizAnswer>

## 問3

暗号化と復号に同じ鍵を使用する暗号方式はどれか。

1. 公開鍵暗号方式
2. 共通鍵暗号方式
3. ハイブリッド暗号方式
4. ハッシュ関数方式


<QuizAnswer>

**答え: 2**

共通鍵暗号方式は、暗号化と復号に同じ鍵（共通鍵）を使用する方式です。代表的なアルゴリズムにAESがあります。公開鍵暗号方式は暗号化と復号に異なる鍵（公開鍵と秘密鍵）を使用する方式であり、共通鍵暗号方式とは仕組みが異なります。共通鍵暗号方式は処理速度が速い反面、通信相手ごとに異なる鍵が必要になるという特徴があります。

</QuizAnswer>

## 問4

公開鍵暗号方式において、送信者が受信者に暗号文を送る場合、送信者が暗号化に使用する鍵はどれか。

1. 送信者の公開鍵
2. 送信者の秘密鍵
3. 受信者の公開鍵
4. 受信者の秘密鍵


<QuizAnswer>

**答え: 3**

公開鍵暗号方式で暗号文を送る場合、送信者は受信者の公開鍵で暗号化します。受信者は自分の秘密鍵で復号します。公開鍵は誰でも入手できますが、対応する秘密鍵を持つ受信者だけが復号できるため、機密性が確保されます。

</QuizAnswer>

## 問5

多要素認証の説明として、最も適切なものはどれか。

1. IDとパスワードに加え、秘密の質問にも回答させる認証方式
2. 知識・所持・生体の3つの認証要素のうち、2つ以上を組み合わせて認証する方式
3. パスワードを一定時間ごとに自動変更する認証方式
4. 複数のWebサービスに同じIDでログインできる認証方式


<QuizAnswer>

**答え: 2**

多要素認証とは、「知識情報（パスワードなど）」「所持情報（ICカード、スマートフォンなど）」「生体情報（指紋、顔など）」の3つの認証要素のうち、2つ以上を組み合わせて本人確認を行う方式です。選択肢1はIDとパスワードも秘密の質問もともに「知識情報」であるため、多要素認証には該当しません。

</QuizAnswer>

## 問6

ネットワーク上で外部からの不正アクセスを防ぐために、内部ネットワークと外部ネットワークの間に設置し、通信を制御する仕組みはどれか。

1. VPN
2. ファイアウォール
3. IDS
4. プロキシサーバ


<QuizAnswer>

**答え: 2**

ファイアウォールは、内部ネットワークと外部ネットワーク（インターネット）の境界に設置し、あらかじめ定めたルールに基づいて通信の許可・遮断を行うセキュリティ機器です。VPNは暗号化された仮想的な通信経路を構築する技術、IDSは不正侵入を検知するシステムであり、通信制御そのものを行うファイアウォールとは役割が異なります。

</QuizAnswer>

## 問7

ISMS（情報セキュリティマネジメントシステム）の説明として、最も適切なものはどれか。

1. 情報セキュリティ製品の技術的な認証制度である
2. 組織が情報セキュリティを管理するための仕組みを継続的に運用・改善する枠組みである
3. ウイルス対策ソフトの性能を評価する国際規格である
4. ネットワーク機器のセキュリティ設定を自動化するツールである


<QuizAnswer>

**答え: 2**

ISMS（Information Security Management System）は、組織が情報資産のセキュリティを体系的に管理・運用し、継続的に改善していくための枠組みです。JIS Q 27001（ISO/IEC 27001）に基づき、リスクアセスメントを行い、情報の機密性・完全性・可用性をバランスよく維持・改善することを目的としています。一過性の活動ではなく、PDCAサイクルによる継続的な改善が特徴です。

</QuizAnswer>

## 問8

指紋や静脈パターンなど、身体的な特徴を利用して本人確認を行う認証方式はどれか。

1. ワンタイムパスワード認証
2. チャレンジレスポンス認証
3. 生体認証（バイオメトリクス認証）
4. シングルサインオン


<QuizAnswer>

**答え: 3**

生体認証（バイオメトリクス認証）は、指紋、虹彩、静脈パターン、顔などの身体的特徴を利用して本人確認を行う認証方式です。パスワードのように忘れたり盗まれたりするリスクが低い一方、一度漏えいすると変更が難しいという特徴があります。ワンタイムパスワードは一度限り有効なパスワードを用いる方式、シングルサインオンは一度の認証で複数サービスを利用できる仕組みです。

</QuizAnswer>

## 問9

フィッシング攻撃の説明として、最も適切なものはどれか。

1. 大量のデータを送りつけてサーバを停止させる攻撃
2. 実在する組織を装った偽のメールやWebサイトで個人情報を詐取する攻撃
3. ネットワーク上の通信データを盗聴する攻撃
4. ソフトウェアの脆弱性を悪用して管理者権限を奪う攻撃


<QuizAnswer>

**答え: 2**

フィッシング（Phishing）は、金融機関や有名企業などを装った偽のメールやWebサイトを用いて、IDやパスワード、クレジットカード番号などの個人情報を不正に詐取する攻撃です。選択肢1はDoS攻撃、選択肢3は盗聴（スニッフィング）、選択肢4は脆弱性攻撃の説明です。

</QuizAnswer>

## 問10

リスク対応の方法のうち、「リスク回避」に該当するものはどれか。

1. 情報漏えいのリスクに備えてサイバー保険に加入する
2. リスクのある業務そのものを廃止する
3. セキュリティ対策ソフトを導入してリスクを低減する
4. リスクが小さいと判断し、特に対策を講じずに受け入れる


<QuizAnswer>

**答え: 2**

リスク回避とは、リスクの原因となる活動や業務そのものを取りやめることで、リスクが発生する可能性をなくす対応策です。選択肢1はリスク移転（転嫁）、選択肢3はリスク低減（軽減）、選択肢4はリスク受容（保有）に該当します。リスク管理ではこれら4つの対応策を状況に応じて使い分けることが重要です。

</QuizAnswer>
