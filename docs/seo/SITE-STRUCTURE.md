## 内部リンク設計

### リンクルール

#### ルール1: クラスター内相互リンク
- 同一資格の記事は **すべて相互リンク** する
- ハブ記事（概要記事）から他の全記事へリンク
- 各記事からハブ記事へ必ずリンク

#### ルール2: クラスター → ピラー
- 全クラスター記事から、関連するピラーコンテンツへリンク
- 例: 薬膳コーディネーター記事 → 「食の資格おすすめ10選」

#### ルール3: ピラー → クラスター
- ピラーコンテンツから、各資格のハブ記事へリンク
- 例: 「食の資格10選」→ 各食関連資格の概要記事

#### ルール4: 関連クラスター間クロスリンク
- 同カテゴリの異なる資格間でクロスリンク
- 例: 薬膳コーディネーター ←→ 食生活アドバイザー
- 例: カラーコーディネーター ←→ 色彩検定

#### ルール5: 記事 → クイズ
- 資格記事から関連クイズページへリンク（既に実装済み）

### サイドバー構成

サイドバー:

## ページタイプ別SEO設定

### 記事ページ (/[slug])
| 要素 | 設定 |
|-----|------|
| title | `{記事タイトル} | シカクのミカタ` |
| description | frontmatter の description |
| canonical | `https://shikaku-no-mikata.com/{slug}` |

## 将来の拡張案

### Phase 2以降で検討するページ
```
├── /author/[name]           # 著者プロフィールページ（E-E-A-T）
├── /compare/[a]-vs-[b]     # 資格 vs 資格 比較ページ

```

### URL設計の原則
1. **日本語スラッグは避ける** — ローマ字表記で統一（現状維持）
2. **フラットURL構造** — `/[slug]` で記事は全てルート直下（現状維持）
3. **意味のあるスラッグ** — `[資格名]-[記事タイプ]` の命名規則
4. **301リダイレクト管理** — URL変更時は必ずリダイレクト設定
